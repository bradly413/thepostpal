## 🔴 Critical — ✅ ALL FIXED (`11e5325`, Claude, browser-verified: native dims, as-post 105px clear, X publish blocked, sched row clear, mobile 266px fits)

Fix: keep the native image in state; produce the exact-size copy at publish/download/platform-switch time only; never downscale above-target sources.
- **R2 Post preview slides under the prompt bar** — `.frame-wrap.as-post` still centers on the old 132px reserve with `max-height: calc(100% - 150px)` (S:2644-2645) while the bar moved to bottom:96px. Fix: share one centerline var + size as-post against the lifted bar.
- **R3 TikTok silently publishes to Instagram** — tiktok is `publishable:false`/"preview only" yet `publish()` maps tiktok→instagram (S:1154-1159) and `socialPlatformsFromComposerId` maps tiktok→["instagram"] — a 9:16 TikTok-framed post quietly ships to the IG feed. Trust violation. Fix: block publish for non-publishable platforms with clear copy + a "switch to Instagram/Facebook" action.
- **R4 Mobile is broken** — (a) the white-room `bottom:96px` bar override is appended after the ≤600px rule, killing mobile positioning; (b) intent strip: 7×50px fixed items clip in the ≤600px rail (VERIFIED at 390px); (c) ≤600px `.frame-wrap` width clamp is dead vs inline frameWrapStyle px. Fix: scope overrides, shrink icons at ≤600px, make frameWrapStyle viewport-aware.
- **R5 Video flow degraded** — (a) no platform UI in video-idle (chip renders only when work exists; old rail always showed) → video composes against stale aspect; (b) Enter in the input fires the IMAGE pipeline and unmounts VideoComposer mid-config (S:1703 — button is guarded :1880, Enter isn't); (c) Edit/Preview toggle on a done video renders the video URL as a CSS background-image → blank frame + image tools on a video. Fix: show chip in video mode, guard Enter, force template for mediaKind=video.
- **R6 Picks carry the previous image's state** — gen-stack click, gallery onPick, and upload onUploaded (S:1357, :1656, :1327) reset nothing: stale captionText/captionTags can PUBLISH with the new image; stale edit pan/zoom/filters apply; stale CaptionVariantPicker stays mounted; clickable mid-generation → in-flight compose overwrites the pick (no guard/abort). Fix: shared `adoptImage(url)` that resets caption/edit/aiCaptionRef and aborts in-flight generation.
- **R7 Inference stomps the user's explicit platform choice** — compose fallback `pIdx = idx >= 0 ? idx : 0` (S:965-967): pick TikTok in the chip, type "a post about our sale" → silently switched to Instagram. Fix: fall back to current platformIdx; explicit chip choice should outrank inference (or at least non-statements).
- **R8 Scheduling unreachable from the studio** — `setWhen` never called (S:205); the entire schedule branch (S:1116-1152), date/time row, and "Schedule" titles are dead. (Pre-existing since `8491bb0`, but the redesign removed its natural home.) Fix: add a Now/Schedule control to the done-state UI (e.g. next to Publish) or delete the branch and route through calendar deliberately.

## 🟠 High — H1 ✅ (`1d61805`), H2/H3/H4 ✅ (`13f0d65`), M4 ✅. H5-H8 ✅ ALL FIXED

- **H1 [security] CSS injection via posts.mediaUrl** — `/api/posts` stores any string; studio seeds history from it (`startsWith("http")`) and interpolates into CSS `url('…')` (gallery :89, gen-stack S:1366, ambient-glow S:1382). In multi-user Command tenants a member can inject CSS into an admin's session. Fix: validate `new URL(mediaUrl).protocol ∈ {http,https}` + reject quote/paren/backslash/whitespace at write; same for `/api/photos` url (L3).
- **H2 [a11y] History gallery: no focus trap + window-level Enter** — Tab escapes behind the modal (useFocusTrap exists, unused here); Enter anywhere fires onPick (Close button double-activates; Enter on a side card opens the CENTER card); active index shifts when a generation lands while open; arrow keys bind to window. Fix: useFocusTrap, scope key handling, real buttons with own activation, track active by key.
- **H3 [a11y] New contrast failures** — intent icons rest `#b6b4b0` = 1.92:1 (needs 3:1 — the "dim" went too dim); coverflow `.cf-date` `#76767e` fails on all four gradient cards; pagination bullets 2.13:1 + 8px targets; TAB keycap 2.96:1; photos sort `opacity-60` 4.28:1; gallery chips ~3.07:1 on bright images; gen-stack focus ring diluted by card opacity. Fix: darken (#8a8884+ icons, ink dates, 0.55+ bullets, 0.66 keycap), opacity:1 on focused gs-card.
- **H4 [a11y] Ghost autofill invisible to screen readers** — Tab is hijacked with no announcement (aria-hidden ghost, no live region). Fix: visually-hidden `aria-live="polite"` "Suggestion: … — press Tab to accept" + aria-describedby hint.
- **H5 [react] Side effect inside setGenHistory updater** — pushHistory calls saveGenerationToMedia (fetch + ref mutation) inside the updater (S:323-332); StrictMode double-invocation safe only by accident. Fix: archive from an effect on genHistory or at call site.
- **H6 [perf] Raster/memory blowout** — up to 14 base64 JPEGs in state (~5-8MB strings); gallery paints ALL cards incl. opacity-0 (~100MB+ decoded peak with stack + glow). Fix: swap archived entries to their hosted URL after upload; don't render cards with abs>3; consider object URLs.
- **H7 [robustness] handleCropToFrame uncaught SecurityError** — no try/catch + no crossOrigin on remote (S3) images reachable via gallery picks/uploads/?mediaUrl (S:748-775). Crop silently dies. Fix: try/catch + setError + `img.crossOrigin="anonymous"` (+ bucket CORS).
- **H8 [a11y] Coverflow clones tabbable** — loop clones' "Create Post" CTAs all in tab order, occluded slides included. Fix: watchSlidesProgress + inert/tabIndex=-1 on non-visible slides.
