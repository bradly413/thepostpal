## 🔴 Critical — ✅ ALL DONE (Wave 1, 2026-06-11)
Committed: `b7bb086` (C1,C4,C5,C6,C7 — Cursor), `d06704b` (C2,C3 — Codex), `b0ff1c6` (C8 — Claude). Verified: SSRF blocked (generate-image 400 + elevate 502), brand page no crash, tsc clean. Not pushed.


- **C1 [cursor] Brand page crashes on location switch** — `src/app/dashboard/brand/page.tsx:208`. `useTokens(palette)` is called after early returns (`:165`,`:173`); switching to a location with no brand book flips `book→null`, the guard returns early → React "rendered fewer hooks" white-screen. Fix: hoist `useTokens`/its `useMemo` above all guards (`useTokens(palette ?? FALLBACK)`).
- **C2 [codex] SSRF via user-supplied image URL** — `src/app/api/studio/elevate/route.ts:52-63` (`fetchImageAsBase64`). Validation is only `^https?://` + length; no private-IP block exists anywhere. Auth'd tenant can make the server fetch `http://169.254.169.254/...`, `localhost`, private ranges. Fix: shared SSRF-safe fetch util (scheme allowlist, DNS-resolve + reject loopback/link-local/private/reserved, `redirect:"manual"` re-validate, size/time caps); use it here and audit `generate-image` `referenceImage` / any other URL fetch.
- **C3 [codex] Forgeable sessions — hardcoded JWT fallback secret, no prod guard** — `src/lib/auth.ts:24` + `src/proxy.ts:13` default to `"posterboy-dev-fallback-secret-change-me"`. Fix: throw at startup if no `AUTH_SECRET`/`JWT_SECRET`/`NEXTAUTH_SECRET` when `NODE_ENV==="production"`.
- **C4 [cursor] Studio scheduling: throws before guard + wrong timezone** — `PosterboyStudio.tsx:841-845`. `.toISOString()` runs before the `isNaN` check (throws first); the `${date}T${time}:00` string has no TZ → wrong absolute publish time. Fix: validate `Date.parse` first; pin/normalize timezone.
- **C5 [cursor] `searchParams` effect clobbers in-progress work** — `PosterboyStudio.tsx:313-325`. Re-render with `?mediaUrl=` still present overwrites a freshly generated image + resets editor. Fix: run once via ref guard; strip the param after consuming.
- **C6 [cursor] Studio hangs forever in "Generating…"** — `PosterboyStudio.tsx:679-686` + guard `:539`. When `/api/studio/compose` fails with a structured intent (empty `prompt`), the fallback `generate()` early-returns; `genState` stays `"generating"`, no error shows → hard hang until reload. Fix: pass the brief into `generate(briefText)` and reset `genState` on early return.
- **C7 [cursor] Editing a calendar post wipes its image** — `calendar/page.tsx:271,414-416` → `api/posts/[id]/route.ts:108-121`. `openEditPost` doesn't load existing media; save sends `mediaUrl: null`, which the PUT treats as "clear" — any caption/time edit silently deletes the image. Fix: seed the form with the record's media, or omit media keys when unchanged.
- **C8 [claude] Scheduled studio posts can't publish** — `PosterboyStudio.tsx:849-858`. Stores the raw base64 `generatedUrl` in `scheduledPost.mediaUrl`; cron publish passes it to Meta (`meta-api.ts:157`), which can't fetch a `data:` URI → every scheduled studio post fails + bloats the table. Fix: resolve the S3 public URL once (`resolvePublicImageUrl`) and store/publish that. (Claude owns — touches the publish pipeline.)
